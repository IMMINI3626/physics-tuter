# PhysiClinic — 추가 구현사항 개발 가이드

## 개요

본 문서는 PhysiClinic의 현재 구현(키워드 추출 → 문제 생성 → 채점 → 피드백) 위에 추가된
두 가지 핵심 기능의 **살아있는 설계 기준**과, 아직 열려있는 사항을 담는다.

- **(1) 소단원 기반 레벨 시스템 + 교정 루프**
- **(2) 마이페이지 단원별 학습 현황 시각화**

두 기능 모두 구현이 완료되었으며, 상세 스펙·구현 과정에서 발견된 이슈·시딩 이력 등
히스토리성 기록은 [CHANGELOG.md](CHANGELOG.md)로 옮겼다. 이 문서에는 코드가 지금도
참조하는 살아있는 기준(UNIT_MAP)과, 아직 결론이 안 난 사항만 남긴다.

---

## 공통 — 소단원 분류

레벨 시스템은 소단원 단위로 관리하고, 마이페이지에서는 대단원으로 묶어서 시각화한다.
소단원-대단원 매핑은 Firestore가 아닌 프론트 상수로 관리한다.

`public/js/app.js`의 UNIT_MAP 상수 (실제 코드와 동기화된 현재 값):

```js
const UNIT_MAP = {
  '힘과 운동': {
    chapterId: '1',
    subUnits: ['물체의 운동', '뉴턴 운동 법칙', '운동량과 충격량']
  },
  '에너지': {
    chapterId: '2',
    subUnits: ['역학적 에너지 보존', '열역학 법칙', '특수 상대성 이론']
  },
  '전기와 자기': {
    chapterId: '3',
    subUnits: ['원자 모형과 전기력', '에너지 띠와 반도체', '전류의 자기 작용', '전자기 유도']
  },
  '파동': {
    chapterId: '4',
    subUnits: ['파동의 진동과 굴절', '파동의 간섭', '빛의 이중성', '물질의 이중성']
  },
};

// 소단원 → 대단원 찾기 헬퍼
function getChapter(unitName) {
  for (const [chapter, data] of Object.entries(UNIT_MAP)) {
    if (data.subUnits.includes(unitName)) return chapter;
  }
  return null;
}
```

> ⚠️ 이 표는 `public/js/app.js`의 실제 값을 그대로 옮긴 것이다. 소단원을 추가/변경하면
> 반드시 이 문서도 같이 갱신할 것 — `extractKeywords` 프롬프트(`functions/index.js`)의
> 소단원 리스트도 이 목록과 동일하게 유지해야 Gemini 반환값이 UNIT_MAP 키와 어긋나지 않는다.

## 미결 사항

- **Level 3 풀이 과정 채점 정확도** — 실제 구현 후 확인하며 필요 시 B안(AI가 풀이 과정은
  채점하지 않고 피드백만 제공)으로 교체
- **레벨 시스템 도입 이전 세션의 `unitProgress` 백필** — 과거 세션을 근거로 소급 생성하는
  방안을 검토했으나, 그 시절엔 레벨 개념이 없어 정확한 레벨을 복원할 수 없어 보류하기로 함
  (사용자 확인, 2026-07). 필요해지면 세션 개수·최고점·마지막 학습일만 채우고 레벨은 1로
  시작하는 방식으로 재검토.
- **소단원 상세의 점수 추이 그래프가 "다시 풀기" 재도전을 이력 목록과 다르게 취급** — 그래프는
  재도전까지 전부 개별 점으로 찍고, 바로 아래 이력 목록은 같은 문제 재도전을 그룹으로 묶어서
  보여줌. 각자 목적(추이 확인 vs 정리해서 보기)에 맞게 의도적으로 다르게 처리한 것이라 버그는
  아니지만, 통일해달라는 요청이 오면 그래프도 문제 단위(그룹당 최신 점수 하나)로 그리도록
  변경 검토. 상세 내용은 [CHANGELOG.md의 3-4절](CHANGELOG.md#3-4-마이페이지-이력--레벨-배지--같은-문제끼리-그룹핑) 참고.
